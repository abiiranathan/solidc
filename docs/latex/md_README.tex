 Solidc is a robust collection of general purpose header-\/only C libraries and data structures designed for rapid C development.\hypertarget{md_README_autotoc_md1}{}\doxysection{\texorpdfstring{Features}{Features}}\label{md_README_autotoc_md1}
Solidc is designed to be a lightweight, easy-\/to-\/use, and efficient library for C developers. It includes a variety of features\+:

All the functions are implemented in a way that they are safe and easy to use and are fully tested with google test.

Available Headers\+:


\begin{DoxyItemize}
\item \href{solidc.h}{\texttt{ solidc.\+h}} -\/ The main header file that includes all the other headers.
\item \href{os.h}{\texttt{ os.\+h}} -\/ \doxylink{structFile}{File} I/O operations, threads, threadpool, pipes, filepath, and directory operations.
\item \href{list.h}{\texttt{ list.\+h}} -\/ A doubly linked list implementation.
\item \href{slist.h}{\texttt{ slist.\+h}} -\/ A singly linked list implementation.
\item \href{log.h}{\texttt{ log.\+h}} -\/ A simple logging library.
\item \href{map.h}{\texttt{ map.\+h}} -\/ A hash map implementation.
\item \href{ordered_map.h}{\texttt{ ordered\+\_\+map.\+h}} -\/ An ordered map implementation.
\item \href{socket.h}{\texttt{ socket.\+h}} -\/ A simple socket library.
\item \href{str.h}{\texttt{ str.\+h}} -\/ A string library with a variety of string manipulation functions(al most all functions are implemented in a way that they are safe to use).
\item \href{trie.h}{\texttt{ trie.\+h}} -\/ A trie implementation for strings.
\item \href{vec.h}{\texttt{ vec.\+h}} -\/ A dynamic array(vector) implementation.
\item \href{set.h}{\texttt{ set.\+h}} -\/ Defines {\ttfamily SET\+\_\+\+DEFINE} macro to define a type-\/safe set.
\end{DoxyItemize}

\hypertarget{md_README_autotoc_md2}{}\doxysection{\texorpdfstring{Usage}{Usage}}\label{md_README_autotoc_md2}
Solidc is a header-\/only library, so you can include the header files directly in your project.

Clone the repository with the following command\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{git\ clone\ https://github.com/abiiranathan/solidc.git}

\end{DoxyCode}


Include the header files in your project or install system-\/wide with cmake.


\begin{DoxyCode}{0}
\DoxyCodeLine{mkdir\ build\ \&\&\ cd\ build}
\DoxyCodeLine{cmake\ ..}
\DoxyCodeLine{sudo\ cmake\ -\/-\/install\ .\ -\/-\/prefix=/usr/include}

\end{DoxyCode}



\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{preprocessor}{\#include\ <solidc/trie.h>}}
\DoxyCodeLine{\textcolor{preprocessor}{\#include\ <solidc/string.h>}}

\end{DoxyCode}


Build your project and link with the math library(-\/lm).\hypertarget{md_README_autotoc_md3}{}\doxysection{\texorpdfstring{Testing}{Testing}}\label{md_README_autotoc_md3}
All functions are tested with google test. Install google test with the following command\+:

Ubuntu\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{sudo\ apt-\/get\ install\ libgtest-\/dev}

\end{DoxyCode}


Arch Linux\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{sudo\ pacman\ -\/S\ gtest}

\end{DoxyCode}


Run the tests with the following command\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{make\ test}

\end{DoxyCode}
